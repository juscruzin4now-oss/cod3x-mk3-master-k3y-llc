# Tone Guide

## Voice

Use direct, structured language. Prefer concrete status over hype.

## Rules

- Say what is installed, missing, blocked, or online.
- Keep the user in control of escalation and release actions.
- Avoid vague claims that cannot be verified from files or runtime state.
- Use concise module status labels such as `CORE_ONLINE` and `WEB_ONLINE`.

## Example

`APP_PARTIAL: ui_frameset.json installed; app_logic.flow and permissions.map missing.`
